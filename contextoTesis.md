3. Contexto del porque se realiza el proyecto:
    El proyecto nace de la necesidad de reemplazar el sistema de ficha anterior, que presentaba carencias críticas en la gestión de emergencias. El sistema anterior fue desarrollado por un estudiante de administración sin conocimientos de programación, lo que resultó en una herramienta poco funcional y segura. Carecía de módulos para el registro de eventos, seguimiento de recursos y generación de informes, lo que dificultaba el trabajo de los departamentos de despacho y operaciones. por eso se me asigno la tarea de crear un nuevo sistema de ficha para reemplazar el que existia y asi poder llevar un mejor control de los eventos que se presentaban en la empresa. el proyecto fue desarrollado utilizando PHP, MySQL, HTML, CSS y JavaScript, con un enfoque en la usabilidad y la eficiencia.
4. contexto uso del sistema anterior
    El sistema anterior funcionaba como una ficha que se llenaba y se mostraba todo en una sola vista, no tenia modulos entonces los despachadores de noche tenian que gritar en el departamento de operaciones donde se atiende la llamada para saber si habia fichas creadas, ademas, las fichas se podian eliminar por completo, el rol administrador se podia eliminar asi mismo, existia saturacion de datos ya que no contaba con una administracion de los datos y ram del servidor y se sobrecargaba en el buscador de fichas y se volvia lento y poco funcional. por lo cual el sistema era poco funcional y poco seguro.  
5. objetivos del nuevo sistema
    Objetivo general: implementar un sistema web de gestión de emergencias que permita mejorar la eficiencia y seguridad en el manejo de incidentes reportados al VEN 911.
    Objetivos especificos: 
    * Levantar los requerimientos funcionales y operativos de los departamentos de despacho y operaciones para subsanar las carencias críticas del software anterior. 
    * Diseñar una arquitectura de software limpia y una base de datos relacional normalizada que garantice la integridad y persistencia de la información.
    * desarrollar un sistema de gestión de emergencias que permita mejorar la eficiencia y seguridad en el manejo de incidentes reportados al VEN 911.
